# Portfolio Site Prompt

Build a polished portfolio site for a public library of AI-native operating skills.

Use the repo's `skills.json` as the content source. The site should feel like a practical library of agent workflows, not a prompt-engineering hobby page.

## Audience

- Corp Dev and strategy executives
- Founders and operators
- AI-native teams evaluating how real workflows become agentic systems
- Recruiters, hiring managers, and collaborators who want proof of applied judgment

## Site Structure

- Homepage with a concise positioning statement.
- Skill grid with one card per skill.
- Detail page or expandable detail view for each skill.
- GitHub link for every skill.
- Clear CTA to view the full repo.

## Detail Page Sections

Each skill should show:

- Problem it solves
- How it works
- Tools and systems it touches
- Example use case
- Build notes or design rationale
- Link to the GitHub folder

## Visual Direction

Quiet, sharp, and credible. More operator system than SaaS landing page.

Use restrained typography, dense-but-readable cards, strong hierarchy, and a professional palette. Avoid purple AI gradients, floating orbs, cartoon agents, generic robot imagery, and marketing fluff.

## Copy Direction

Lead with the point. Keep the writing specific and compressed.

Position the work as a portfolio of applied judgment systems: repeatable workflows that combine instructions, tools, evals, and operator taste.
